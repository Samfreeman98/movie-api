This web application offers access to a database of information about horror movies.

Business Logic URL HTTP method Request body data format Response body data format
Return a list of ALL movies to user /movies GET None A JSON object holding data about all movies
Return data about a single movie by title to the user /movies/[title] GET None A JSON object holding data about a single movie, containing title and director. Ex: { "Title":"Scream 5", "Director": { "Name":"Matt Bettinelli-Opin, Tyler Gillett", }}, { "Title":"The Conjuring 2", "Director":{ "Name": "James Wan", }}
Return data about a director by name /movies/directors/[directorName] GET None A JSON object holding data about a single director, containing director name. Ex: { "directorName": "James Wan", }
Allow new users to register /users POST A JSON object holding data about the user to add Ex: { "name": "Spencer", "favoriteMovies": [] } A JSON object holding data about the user that was added, including an ID. Ex: { "name": "Matthew", "favoriteMovies": [], "id": "903da19e3-259b-460c-bd6f-34ae838876b0" }
Allow users to update their user info /users/[id] PUT A JSON object holding data about the user which needs to be updated: { "name": "Spencer", "favoriteMovies": [] } A JSON object holding data about the updated user information. Ex: { "id": 2, "name": "Spencer", "favoriteMovies": [] }
Allow users to add a movie to their list of favorites /users/[id]/[movieTitle] POST None Text message indicating whether the user added the movie successfully.
Allow users to remove a movie from their list of favorites /users/[id]/[movieTitle] DELETE None Text message indicating whether the user removed the movie successfully.
Allow existing users to deregister /users/[id] DELETE None Text message indicating whether the user deregister successfully.

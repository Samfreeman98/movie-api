This web application offers users access to a database of information about a variety of movies. Users will be able to register, update account information, and add movies to a list of their favorites

Endpoint URL HTTP method Request body data format Response body data format
Return a list of ALL movies to user /movies GET None A JSON object holding data about all movies
Return data about a single movie by title to the user /movies/title/:title GET None A JSON object holding data about a single movie, containing title, director, genre, and whether it is featured or not. Ex: { "Title": "Inception", "Description": "A thief who steals corporate secrets through the use of dream-sharing technology is given the inverse task of planting an idea into the mind of a CEO.", "genre": { "Name": "Science Fiction", "Description": "Exploring speculative concepts often involving advanced technology, space exploration, and extraterrestrial life." }, "Director": { "Name": "Christopher Nolan", "Bio": "Christopher Nolan is a British-American film director, producer, and screenwriter. He is known for his distinctive filmmaking style and nonlinear storytelling.", "Birth": “1970”, }, "Featured": true }
Return data about a director by name /movies/directors/:Director GET None A JSON object holding data about a single director, containing director name, bio, and birthyear. Ex: "Director": { "Name": "Steven Spielberg", "Bio": "Steven Spielberg is known for his contributions to the adventure and science fiction genres.", "Birth": "1946" }
Return data about a Genre by name /movies/genre/:Genre GET None A JSON object holding data about a single genre, containing a genre name and description. Ex: "Genre": { "Name": "Crime", "Description": "Focuses on criminal activities, law enforcement, and the justice system." }
Allow new users to register /users POST A JSON object holding data about the user to add including username, password, email, and birthday Ex: { "Username": "Eric", "Password": "Ericspassword123", "Email": "Eric@fakeemail.com", "Birthday": "1995-04-06" } A JSON object holding data about the user that was added, including an ID. Ex: { "Username": "Eric", "Password": "Ericspassword123", "Email": "Eric@fakeemail.com", "Birthday": "1995-04-06T00:00:00.000Z", "FavoriteMovies": [], "_id": "65693791e808bf6e46df7d7d", }
Allow users to update their user info /users/:Username PUT A JSON object holding data about the user which needs to be updated: { "Username": "EricNeff", "Password": "Ericspassword555", "Email": "Eric@fakeemail.com", "Birthday": "1985-06-02" } A JSON object holding data about the updated user information such. Ex: { "_id": "65693791e808bf6e46df7d7d", "Username": "EricNeff", "Password": "Ericspassword555", "Email": "Eric@fakeemail.com", "Birthday": "1985-06-02T00:00:00.000Z", "FavoriteMovies": [] }
Allow users to add a movie to their list of favorites /users/:Username/movies/:MovieId POST None Message indicating whether the user added the movie successfully.
Allow users to remove a movie from their list of favorites /users/:Username/movies/:MovieId DELETE None Message indicating whether the user removed the movie successfully.
Allow existing users to deregister /users/:Username DELETE None Message indicating whether the user deregister successfully.
